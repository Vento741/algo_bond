## 1. Обновление существующих секций

```tsx

// Стало:
```

### Stats и Features — без изменений

## 2. Новые секции

### 2.1 How It Works

**Позиция:** После Features, перед Access Request Form.

**Дизайн:** 3 шага в горизонтальной линии с коннекторами.

```
┌─────────────┐     ───→     ┌─────────────┐     ───→     ┌─────────────┐
│     ①       │              │     ②       │              │     ③       │
│  Запросите  │              │  Настройте  │              │  Запустите  │
│   доступ    │              │  стратегию  │              │    бота     │
└─────────────┘              └─────────────┘              └─────────────┘
```

**Контент:**

| Шаг | Заголовок | Описание | Иконка (lucide) |
|-----|-----------|----------|-----------------|
| 1 | Запросите доступ | Оставьте заявку с вашим Telegram. Мы отправим персональный инвайт-код | `MessageCircle` |
| 2 | Настройте стратегию | Выберите ML-стратегию, задайте параметры и протестируйте на истории | `Settings` |
| 3 | Запустите бота | Подключите API Bybit и запустите автоматическую торговлю | `Rocket` |

**Визуал:**
- Каждый шаг — карточка с номером (крупный, gold gradient), иконкой, заголовком, описанием
- Между шагами — dashed-line коннекторы (SVG или border-dashed)
- На мобильном: вертикально, коннекторы вниз
- Анимация: `fade-up` с stagger (0.1s, 0.2s, 0.3s)
- Фон секции: `bg-brand-bg` с subtle gradient

### 2.2 Access Request Form

**Позиция:** После How It Works.

**ID:** `#access-request` (для smooth scroll из Hero CTA)

**Дизайн:**

```
┌──────────────────────────────────────────┐
│        🔑 Запросите доступ               │
│                                          │
│  Оставьте ваш Telegram — мы отправим    │
│  персональный код приглашения            │
│                                          │
│  ┌──────────────────────┐  ┌──────────┐ │
│  │ @username             │  │ Отправить│ │
│  └──────────────────────┘  └──────────┘ │
│                                          │
│  Нажимая "Отправить", вы соглашаетесь   │
│  с Условиями использования              │
└──────────────────────────────────────────┘
```

**Поведение:**
- Input: Telegram username (валидация: начинается с @, 5-32 символа, [a-zA-Z0-9_])
- Button: "Отправить" (premium variant)
- Success state: скрыть форму, показать "Заявка отправлена! Мы свяжемся с вами в Telegram" + CheckCircle icon (green)
- Error state: показать ошибку под формой
- Rate limit client-side: Не показывать форму если уже отправлена (localStorage `access_request_sent`)
- Rate limit server-side: При 429 ответе показать "Слишком много попыток. Повторите позже."

**API:**
- `POST /api/auth/access-request` — endpoint из SPEC 4
- Body: `{ "telegram": "@username" }`
- Response: `201 Created` или `409 Conflict` (уже отправлена)

**Визуал:**
- Карточка на тёмном фоне с gold glow-border
- Центрирована, `max-w-lg`
- Иконка `Key` (lucide) в заголовке

### 2.3 Performance

**Позиция:** После Access Request Form, перед FAQ.

**Дизайн:**

Две части:
1. **Equity curve chart** — график роста депозита (mock данные)
2. **Метрики** — 4 карточки с ключевыми показателями

**Equity Curve:**
- **Статический SVG** (не lightweight-charts - маркетинговая страница, не нужна интерактивность)
- Данные: hardcoded массив в компоненте

```typescript
interface EquityPoint { month: string; value: number; }
const EQUITY_DATA: EquityPoint[] = [
  { month: 'Янв', value: 10000 },
  { month: 'Фев', value: 14200 },
  { month: 'Мар', value: 18500 },
  // ... 12 месяцев до 81000
  { month: 'Дек', value: 81042 },
];
```

> Позже заменить на данные из `GET /api/backtest/runs/:id/result` (equity_curve field).

- Зелёный gradient fill, тёмный фон
- Ось X: месяцы, Ось Y: USDT

**Метрики:**

| Метрика | Значение | Цвет |
|---------|----------|------|
| Доходность | +710% | `brand-profit` (green) |
| Макс. просадка | -12.3% | `brand-loss` (red) |
| Sharpe Ratio | 2.41 | `brand-premium` (gold) |
| Win Rate | 68.5% | `brand-accent` (blue) |

- Числа: `font-data` (JetBrains Mono)
- Карточки в grid 2x2 на desktop, 1x4 на mobile

**Disclaimer:**
Под секцией мелким текстом: _"Результаты получены на исторических данных (бэктест). Прошлые результаты не гарантируют будущую доходность."_

### 2.4 FAQ

**Позиция:** После Performance, перед Footer.

**Компонент:** Accordion (shadcn/ui)

> Необходимо установить: `npx shadcn-ui@latest add accordion`

**Вопросы:**

| # | Вопрос | Ответ (краткое) |
|---|--------|-----------------|
| 1 | Что такое AlgoBond? | Платформа алгоритмической торговли криптофьючерсами на Bybit с ML-стратегиями |
| 2 | Как получить доступ? | Оставьте Telegram в форме выше, мы отправим персональный инвайт-код |
| 3 | Какие стратегии используются? | Lorentzian KNN Classification — ML-модель, адаптированная под криптовалютные рынки |
| 4 | Нужно ли отдавать доступ к кошельку? | Нет. Вы создаёте API-ключ только с правами на торговлю, без вывода средств |
| 5 | Какие риски? | Торговля криптофьючерсами несёт высокие риски. Подробнее — в разделе Раскрытие рисков |
| 6 | Сколько стоит? | На данный момент доступ бесплатный для участников закрытого тестирования |

**Визуал:**
- Заголовок: "Частые вопросы"
- `max-w-3xl mx-auto`
- Каждый item: border-bottom, chevron-icon (rotate на открытие)
- Анимация открытия: accordion-down keyframe (уже есть в tailwind.config.js)

## 4. Анимации

Все новые секции используют существующий паттерн `fade-up`:

```css
animation: fade-up 0.7s ease-out both;
animation-delay: calc(var(--delay, 0) * 1s);
```

- Каждая секция появляется при скролле (Intersection Observer или animation-delay)
- Stagger: элементы внутри секции с delay 0.1s, 0.15s, 0.2s...
- Accordion: использует существующие keyframes `accordion-down` / `accordion-up`

## 5. Smooth Scroll

Hero CTA "Запросить доступ" → скроллит к `#access-request`:

```tsx
<a
  href="#access-request"
  onClick={(e) => {
    e.preventDefault();
    document.getElementById('access-request')?.scrollIntoView({
      behavior: 'smooth'
    });
  }}
>
  Запросить доступ
</a>
```
