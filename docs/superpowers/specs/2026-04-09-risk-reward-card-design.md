# Risk/Reward Card - Спецификация

## Цель

Отдельная карточка на дашборде бота, отображающая соотношение потенциальной прибыли (до TP) к потенциальному убытку (до SL) для текущей или последней позиции.

## Расположение

Между Stats Strip и Live Position Card на странице BotDetail:

```
Stats Strip (P&L, UNREALIZED, WIN%, TRADES, DD, UPTIME)
────────────────────────────────────────────────────────
>>> Risk/Reward Card <<<
────────────────────────────────────────────────────────
Live Position Card (entry, current, SL, TP, sine wave bar)
```

## Содержимое карточки

### Элементы

| Элемент | Описание | Стиль |
|---------|----------|-------|
| R/R ratio | `1 : X.X` | #FFD700 (gold), bold, 22px, mono |
| Цель TP1 | `+$XX.XX` + цена | Карточка с зелёной рамкой, метка "активный"/"исполнен" |
| Цель TP2 | `+$XX.XX` + цена (если есть) | Карточка приглушённая/"следующий", или активная после TP1 |
| Риск (SL) | `-$XX.XX` + цена | Карточка с красной рамкой |
| R/R бар | Визуальный бар risk/reward | Градиент red->green, маркер TP1 |
| Подпись | "$1 риска -> $X.XX прибыли" | #555, 9px |

## Формулы

```
reward_tp1 = |tp1_price - entry_price| * quantity
reward_tp2 = |tp2_price - entry_price| * quantity   (если tp2 есть)
risk       = |sl - entry_price| * quantity

rr_ratio   = reward_active / risk
             где reward_active = reward_tp1 если !tp1_hit, иначе reward_tp2
```

После исполнения TP1:
- `quantity` уменьшается (partial close)
- Risk и Reward TP2 пересчитываются с новым quantity
- R/R ratio пересчитывается

## Логика отображения

### Открытая позиция
- Live R/R, обновляется при изменении SL/TP (trailing stop двигает SL)
- Данные из `openPosition` (уже есть в компоненте BotDetail)

### Нет открытой позиции
- Показывать R/R последней закрытой позиции
- Бейдж "Последняя сделка" рядом с заголовком
- Только план (цель/риск), без фактического результата

### Single TP (без multi-TP)
- Одна карточка цели + карточка риска
- Блок TP2 не отображается

### Multi-TP, TP1 активен
- TP1: зелёная рамка, метка "активный"
- TP2: приглушённая рамка, метка "следующий"
- R/R считается по TP1

### Multi-TP, TP1 исполнен
- TP1: зачёркнутая цена, метка "исполнен"
- TP2: зелёная рамка, метка "активный"
- R/R считается по TP2 с уменьшенным quantity

## Источники данных

Все данные уже есть в `PositionResponse`:
- `entry_price` - цена входа
- `stop_loss` - стоп-лосс
- `take_profit` - тейк-профит (основной)
- `quantity` - текущее количество
- `tp1_price` - цена TP1 (из indicators_snapshot)
- `tp2_price` - цена TP2 (из indicators_snapshot)
- `tp1_hit` - флаг исполнения TP1
- `side` - LONG/SHORT (влияет на направление расчёта)

Бэкенд изменений не требует.
