### 1.1 Условия использования (`/terms`)

**Файл:** `frontend/src/pages/Terms.tsx`

**Структура документа:**

1. **Общие положения**
   - AlgoBond — платформа алгоритмической торговли
   - Оператор: [ИП/ООО — указать при реализации]
   - Принятие условий при регистрации

2. **Регистрация и доступ**
   - Закрытый доступ по инвайт-кодам
   - Возраст: 18+
   - Один аккаунт на пользователя
   - Ответственность за сохранность учётных данных

3. **Использование платформы**
   - Разрешённые действия: настройка стратегий, запуск ботов, бэктестинг
   - Запрещённые действия: автоматизированный скрейпинг, обход ограничений, передача доступа третьим лицам
   - Пользователь самостоятельно подключает API-ключи биржи

4. **Финансовая ответственность**
   - Платформа НЕ является финансовым советником
   - Все торговые решения — ответственность пользователя
   - Платформа не несёт ответственности за убытки от торговли
   - Исторические результаты не гарантируют будущую доходность

5. **Интеллектуальная собственность**
   - Стратегии, код, дизайн — собственность AlgoBond
   - Запрет копирования алгоритмов

6. **Ограничение ответственности**
   - Платформа предоставляется "как есть"
   - Не гарантируется бесперебойная работа
   - Максимальная ответственность ограничена суммой оплаты за последний месяц

7. **Прекращение доступа**
   - Платформа может заблокировать аккаунт за нарушение условий
   - Пользователь может удалить аккаунт в любой момент

8. **Разрешение споров**
   - Применимое право: Российская Федерация
   - Досудебный порядок (претензия 30 дней)
   - Суд по месту нахождения оператора

9. **Изменение условий**
   - Платформа вправе изменять условия
   - Уведомление через email за 14 дней

### 1.2 Политика конфиденциальности (`/privacy`)

**Файл:** `frontend/src/pages/Privacy.tsx`

**Требования 152-ФЗ:**

1. **Оператор персональных данных**
   - Наименование: [ИП/ООО — заполнить при реализации]
   - Адрес: [указать]
   - ИНН: [указать]
   - Email для обращений: [указать]

2. **Какие данные собираем**
   - Email (регистрация, вход)
   - Имя пользователя (профиль)
   - API-ключи бирж (зашифрованы Fernet, хранятся в PostgreSQL)
   - Telegram username (форма запроса доступа)
   - Данные об использовании (страницы, действия)
   - IP-адрес (логи сервера)
   - Данные торговли (ордера, позиции, P&L — созданы ботами)

3. **Цели обработки**
   - Предоставление доступа к платформе
   - Исполнение торговых операций через API бирж
   - Техническая поддержка
   - Улучшение платформы (аналитика)

4. **Правовые основания**
   - Согласие субъекта (checkbox при регистрации)
   - Исполнение договора (Условия использования)
   - Законный интерес оператора (безопасность, аналитика)

5. **Хранение данных**
   - Срок: на время действия аккаунта + 3 года после удаления
   - Место: серверы на территории Российской Федерации
   - Шифрование API-ключей: Fernet (симметричное шифрование)
   - Пароли: bcrypt hash

6. **Передача третьим лицам**
   - Bybit (API-ключи передаются для исполнения ордеров)
   - Хостинг-провайдер (серверная инфраструктура)
   - НЕ передаём рекламным сетям, НЕ продаём данные

7. **Права субъекта**
   - Доступ к своим данным
   - Исправление неточных данных
   - Удаление данных (удаление аккаунта)
   - Отзыв согласия
   - Обращение: email оператора

8. **Трансграничная передача**
   - API-ключи передаются на серверы Bybit (Сингапур/Гонконг)
   - Пользователь даёт согласие при подключении биржевого аккаунта

9. **Безопасность**
   - HTTPS шифрование
   - JWT авторизация с коротким сроком жизни токенов
   - Fernet шифрование секретов
   - Регулярный аудит

> **Примечание для реализации:** Поля с [указать] - заполнить из `settings.OPERATOR_NAME`, `settings.OPERATOR_INN`, `settings.OPERATOR_ADDRESS`, `settings.OPERATOR_EMAIL` (добавить в `backend/app/config.py`). Если значения не заданы в .env, отображать "Не указано" и оставить `// TODO: заполнить данные оператора` в коде.

### 1.3 Политика обработки данных и cookies (`/cookies`)

**Файл:** `frontend/src/pages/Cookies.tsx`

> Название "Политика обработки данных" вместо "Cookie Policy", т.к. AlgoBond использует localStorage (не HTTP cookies) для JWT-токенов.

**Содержание:**

1. **Что мы храним в браузере**
   - Авторизационные данные сессии (localStorage)
   - Флаг принятия согласия на обработку данных (localStorage)
   - Настройки интерфейса: тема, предпочтения (localStorage)

2. **Cookies третьих сторон**
   - На данный момент: отсутствуют
   - При подключении аналитики (Яндекс.Метрика и т.п.) — будет обновлено

3. **Зачем это нужно**
   - Авторизация в платформе
   - Сохранение настроек между сессиями
   - Безопасность (защита от несанкционированного доступа)

4. **Как управлять**
   - Очистка данных: Настройки браузера → Очистить данные сайта
   - При очистке — потребуется повторный вход

### 1.4 Раскрытие рисков (`/risk`)

**Файл:** `frontend/src/pages/RiskDisclosure.tsx`

**Содержание:**

1. **Общие риски криптовалют**
   - Высокая волатильность
   - Возможность полной потери средств
   - Регуляторные риски (РФ и международные)

2. **Риски фьючерсной торговли**
   - Кредитное плечо увеличивает как прибыль, так и убытки
   - Ликвидация позиций при недостаточной маржи
   - Проскальзывание и гэпы

3. **Риски алгоритмической торговли**
   - Стратегия, прибыльная на истории, может быть убыточной в реале
   - +710% по RIVERUSDT — результат бэктеста, НЕ гарантия
   - Технические сбои (API, интернет, сервер)
   - Модель может не адаптироваться к изменению рынка

4. **Риски платформы**
   - AlgoBond — инструмент, НЕ финансовый советник
   - Платформа не управляет вашими средствами
   - Вы самостоятельно подключаете API-ключи и принимаете торговые решения
   - Платформа не компенсирует убытки от торговли

5. **Рекомендации**
   - Торгуйте только на средства, потерю которых можете позволить
   - Начинайте с demo-режима
   - Диверсифицируйте риски
   - Регулярно проверяйте работу ботов

## 2. Cookie Banner

### Компонент `CookieBanner.tsx`

**Расположение:** `frontend/src/components/CookieBanner.tsx`

**Поведение:**
- Фиксирован внизу экрана (`fixed bottom-0`)
- Показывается один раз, пока пользователь не нажмёт "Принять"
- Состояние сохраняется в `localStorage.getItem('cookie_consent')`
- Если `cookie_consent === 'accepted'` — не показывать
- Появляется с анимацией slide-up

**UI:**
```
┌──────────────────────────────────────────────────────────────────────┐
│ Мы используем cookies и localStorage для работы сервиса.            │
│ Подробнее — [Политика обработки данных]          [Принять]          │
└──────────────────────────────────────────────────────────────────────┘
```

- Фон: `bg-brand-card/95 backdrop-blur-sm`
- Текст: `text-sm text-gray-300`
- Ссылка: `text-brand-premium hover:underline` → `/cookies`
- Кнопка: `variant="premium" size="sm"` → сохраняет в localStorage, скрывает баннер
- Рендерится в `App.tsx` на всех страницах

## 3. Checkbox согласия на регистрации

### Изменения в `Register.tsx`

Добавить перед кнопкой "Зарегистрироваться":

```tsx
<label className="flex items-start gap-3 text-sm text-gray-400 cursor-pointer">
  <input
    type="checkbox"
    required
    checked={consent}
    onChange={(e) => setConsent(e.target.checked)}
    className="mt-1 accent-brand-premium"
  />
  <span>
    Я согласен с{' '}
    <Link to="/terms" className="text-brand-premium hover:underline" target="_blank">
      Условиями использования
    </Link>{' '}
    и{' '}
    <Link to="/privacy" className="text-brand-premium hover:underline" target="_blank">
      Политикой конфиденциальности
    </Link>
  </span>
</label>
```

- `consent` - новый state, по умолчанию `false`
- Кнопка "Зарегистрироваться" `disabled` пока `!consent`
- Валидация: client-side (`required` checkbox) + backend audit trail (`consent_accepted_at` в таблице users)

> **ВАЖНО:** Checkbox согласия реализуется в SPEC 4 (auth-and-error-pages), т.к. SPEC 4 владеет Register.tsx. Здесь только требование.
