# /optimize [SYMBOL] [TF] [FLAGS] — Оптимизация стратегии

Запуск grid search backtesting для поиска оптимальных параметров стратегии.

## Аргументы

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| SYMBOL | RIVERUSDT | Торговая пара |
| TF | 15 | Таймфрейм (1, 5, 15, 60, 240) |

## Флаги

| Флаг | Описание |
|------|----------|
| --quick | Только Фаза 1 (coarse grid) |
| --params GROUP | Оптимизировать конкретную группу: knn, risk, filters, trend, ribbon, smc, flow |
| --start DATE | Начало периода (default: 2025-01-01) |
| --end DATE | Конец периода (default: сегодня) |
| --capital N | Начальный капитал (default: 100) |

## Поведение

1. **Делегировать агенту `optimizer`** — это специализированный агент для оптимизации
2. Агент логинится, запускает серию бэктестов, анализирует результаты
3. Выводит TOP-10 таблицу и сохраняет лучший конфиг

## Примеры

```
/optimize RIVERUSDT 15                          # Полная оптимизация RIVER 15m
/optimize BTCUSDT 5 --quick                     # Быстрая оптимизация BTC 5m
/optimize RIVERUSDT 15 --params risk             # Только параметры риска
/optimize ETHUSDT 60 --start 2025-06-01          # ETH 1h с июня 2025
```

## Шаги выполнения

При получении команды `/optimize`:

1. **Парсинг аргументов** из пользовательского ввода
2. **Запуск агента `optimizer`** через Agent tool с prompt:
   ```
   Оптимизируй стратегию Lorentzian KNN для {SYMBOL} на {TF} таймфрейме.
   Период: {START} → {END}, капитал: {CAPITAL}.
   Режим: {full/quick/params}.
   Выполни grid search и выведи TOP-10 результатов.
   ```
3. **Передать результат** пользователю
